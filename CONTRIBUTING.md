### ğŸ’¬ Commit Message Format

```bash
type(scope): brief description

Detailed description if needed

Fixes #issue_number
```

**Types:**
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `style`: Code style changes
- `refactor`: Code refactoring
- `test`: Adding tests
- `chore`: Maintenance

**Examples:**
```bash
feat(plugin): add new economy integration
fix(map): resolve spawn point positioning issue
docs(readme): update installation instructions
```
